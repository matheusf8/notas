Projeto de Anotações (Backend + Frontend)

Este repositório contém um sistema completo de anotações com autenticação JWT, desenvolvido em FastAPI (backend) e React/Vite (frontend).

🚀 Funcionalidades

Cadastro e login de usuários.

Autenticação via JWT.

CRUD de notas (criar, listar, atualizar, deletar).

Proteção de rotas: apenas usuários logados acessam notas.

Integração entre backend e frontend.

⚙️ Backend

Instalação

cd backend
python -m venv venv
venv\Scripts\activate   # Windows
source venv/bin/activate # Linux/Mac
pip install -r requirements.txt

Executando

uvicorn main:app --reload

Endpoints

Swagger UI: http://127.0.0.1:8000/docs

ReDoc: http://127.0.0.1:8000/redoc

Fluxo de Autenticação

Cadastro: POST /auth/cadastro

Login: POST /auth/login → retorna access_token

Authorize no Swagger: cole apenas o token cru (eyJhbGci...). O Swagger monta automaticamente Authorization: Bearer ....

Rotas de notas: POST /notas/, GET /notas/, PUT /notas/{id}, DELETE /notas/{id}.

⚙️ Frontend

Instalação

cd frontend
npm install

Executando

npm run dev

Fluxo no Frontend

Login.jsx: envia usuário/senha para /auth/login, salva access_token no localStorage.

Principal.jsx: todas as requisições usam:

const token = localStorage.getItem("token");
fetch("http://localhost:8000/notas/", {
  headers: {
    "Authorization": `Bearer ${token}`,
    "Content-Type": "application/json"
  }
});

🛡️ Segurança

Senhas com hash bcrypt.

Tokens JWT expiram em 30 minutos.

Rotas de notas protegidas.

✅ Conclusão

Este projeto integra FastAPI e React para um sistema seguro de anotações. O backend fornece autenticação JWT e rotas protegidas, enquanto o frontend consome essas rotas e gerencia o token automaticamente.
